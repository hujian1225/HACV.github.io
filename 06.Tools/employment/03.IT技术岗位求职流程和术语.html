    # IT技术岗位求职流程和术语

    # 01.术语类

    # 1.1.实习，试用期，正式员工

    • 1、实习期一般薪资很低
    • 2、试用期，一般为正式薪资的0.8
      • 有例外的公司，比如“编程猫”，试用期和正式薪资一样
      • 特斯拉公司在2020年（2021届秋招）说的
    • 3、正式员工

    # 1.2.其他常用语

    # （1）OC（offer call）

    • offer call比面试当场“口头offer”的好
    • offer call：HR打来电话告知面试通过，给口头offer.

    # （2）意向书

    • 通常都是邮件收到意向书，这时候才能算尘埃落定
    • 意向书一般内容：月薪，福利，企业信息，三方签订
    • 也有企业是发短信

    # （3）“云证”

    • 信息核实，类似背景调查
    • 我目前只听到腾讯有这个？似乎就是上传身份证啥的进行验证，这一步一般已经通过HR面了

    # （4）“背景调查”

    • 只记得腾讯的背景调查很严格

    # （5）HC（Headcount）

    俗称人头，稍微专业点讲就是这家公司打算招的人数

    # （6）argue offer

    其实就是在原有offer上讨价还价，希望给更高的薪资

    就是薪资argue的问题？

    • bat级别的基本不用argue了，几乎没听说成功的，除非拿谷歌微软来，但是有谷歌你也不会去bat了 这两天接到了谈薪电话，低于我的预期，但平台都挺好的。跟hr argue了
    • 说有没有要钱的技巧，校招开奖的时候，就是那种能让hr听后感觉很舒服，很愿意给你申请高价offer 别人有个模板，就是先表达你来该公司的强烈意愿，然后表示你拿了什么什么厂的什么价位的offer，白菜就别说了，如果sp可以说，然后表示如果能在这个价位基础之上加多少钱就更合适了，自己未来还是想来这边发展的，希望这边可以帮忙申请一下

    # （7）OD

    • 外包的意思
    • OD一般是指的是华为的“外包”公司，比如像德科这种。网上其实有很多人都吐槽过这个招聘模式，因为招进去的人不直接是华为内部的人，挂在德科名下或其他，工作地点大部分都是在华为公司，虽然明面上同薪同酬，福利奖金与华为内部员工一致，但其实差很多，尤其是待遇和期权上面。

    # （8）mentor

    • mentor，n.导师；指导者；师傅

    # 1.3.企业类型

    “外包”

    是指企业动态地配置自身和其他企业的功能和服务，并利用企业外部的资源为企业内部的生产和经营服务

    # 04.『笔/面试流程和建议』

    # （1）笔试

    笔试编程，如果不是BAT的。 那些中厂

    • 一般，要是5道题。
      • 前3道是不会出现dp的，前三题一般考思维，模拟，贪心
    • 如果是3道题，那么第1题简单，后2题是难的

    # （2）面试

    # 1、简历的重要性

    面试的第一轮，一面所有一般都来源于你的简历或你的信息，所以，你在做自我介绍的时候，最好有引导，不然，可能就会被问面试官题库中的问题

    此外划重点：
    （1）面试的时候，不会就是不会，不用不好意思。素质高的面试官都是会让你不会太尴尬的。此外，技术面试你的，未来还可能是你同事，所以不用担心。比如，笔者某次面试的面试官，后边发我offer后说的是，它是这个组的leader，我如果入职，就是他一对一带我
    （2）简历不要作假，不要夸夸其词，不然面试深入问的时候，你不会的话就会很尴尬 。

    # 2、准备一份自我介绍

    • 自我介绍准备一份：2分钟和3分钟版本的
    • 有的公司会有自我介绍，也有的公司没有自我介绍。

    # 3、HR面

    一般HR面的面试官是不懂技术的，你要做好的准备就是不卑不亢，相互交流。

    防止被HR压薪，自己利益受损 也要防止自己趾高气扬，给HR不好的映像

    # 4、其他术语

    • 简历面：根据你写的简历，问对应的问题，一般是第1轮技术面
    • leader面：一般是第2面
    • 小组面试：只有极少数的公司使用，一般小组面是『非技术』岗位的

    # 05.『校园招聘和社会招聘』

    • 复习方式——对比

    # （1）学习方式

    • ACM有人说过，如果你暂时不能理解某个算法，那就手抄这个，等到沉淀，自然会有提升
    • 1、在学习编程的时候切忌眼高手低，有人在大一初学C语言时，由于天赋较差，每天晚上就对着高手的代码抄写，抄完了就默写，循环往复。
    • 2、等到学期末上机考试的时候，我惊讶地发现自己写代码的速度和规范性比其他同学要强，许多基础性的功能代码基本上“手到码出"。
    • 3、一些参加ACM竞赛的同学告诉我，不断地抄写代码是一个很好的学习方式，这也是为什么很多ACM高手能在一般人还在看题的时候就把代码写出。一句话，无它，唯手熟尔。
    • 4、许多算法在刚接触的时候很难理解，这个时候通过抄写、默写代码，能够让你在某天醍酸灌顶、恍然大悟，但如果你只是用两只眼看着它，永远也学不会。

    # （2）『校园招聘』重基础

    • 秋/春招复习经验

    • 1.准备复习不要用很厚很厚的手册书，除非你之前已经通读过一遍，这点很重要，否则你将会浪费大量时间。

    • 2.别人经验贴里所列参考书籍绝不是短短一两个月看完的，如果以前没看过，请不要盲从。（重点）

    # （3）『社会招聘』重框架底层代码

    • 收集自牛客
    • 如果面试目标岗位需要转语言，那么面试多数问题都是围绕中间件的，redis，kafka，mysql，zk这些要看的比较细一些
    • 渣渣找工作历险记 ： 的确如此

    # 06.面试评级

    • 根据面试评级报一下薪资待遇
    • 可以最后在HR的时候，问一下面试评级
    • 所谓的普通offer，SP，SSP等都是根据这个来的

    注意事项：

    # （1）面试评级

    有的公司：评级不重要，会argue才重要
举例说明：同样水平，二话不说接了你offer，反而没有人家argue完高，甚至本来人家评级还不如你

    2

    # （2）薪资谈判

    • 注意：有的公司：谈薪资不先说自己的，先问别人给的多少？
    • 上述操作是不妥的，应该等他们说明，你才考虑用不用自己已经有的offer去argue

    # 07.应届生自我保护

    • 『技术面试官』给的评级只能决定你能不能拿到意向书
    • 最终定级的是由『HR』定的，HR不懂技术，那如何定级呢？就是看你手上有没有别的Offer，所以拿到一家，再满意也得接着面别的。

    有人的教训

    及时面试官：你不是s就是a
HR：你是b+，因为你没有别的offer
