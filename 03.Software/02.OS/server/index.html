            # 01.项目经验撰写『STAR原则』

            • 背景(Situation)：是在一种什么样的情况下决定做这个项目的，准备解决什么问题;

              • 希望写一个Web服务器来承载，我的个人博客。

            • 目标(Task)：计划完成哪些事项、达成什么样的具体成果;

              • 1、高并发
              • 2、IO多路复用
              • 3、日志

            • 行动(Action)：做了哪些具体工作，遇到了什么样的困难和挑战，是如何解决的;

              • 1、解决粘包
              • 2、进行压力测试

            • 结果(Result):是否完成了最初的任务和目标，带来的商业效果和价值如何。

              • 博客正常运转，达到了XX的并发量
