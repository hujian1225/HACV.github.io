        # （1）知识补充

        词法解析之“贪心法”——贪婪原则： 每个符号应该包含尽可能多的字符，从左到右一个字符一个字符地读入，如果该字符可能组成一个符号，那么再读入下一个字符，判断已经读入的两个字符组成的字符串是否可能是一个符号的组成层部分；如果可能，继续读入下一个字符，重复上述判断。（编译器将程序分解成符号（词法解析））

        记忆口诀和贪婪原则传送门

        # （2）技巧总结

        • 1、进行词法分析（贪婪原则）
        • 2、进行优先级分析（记忆口诀）
        • 3、进行结合性分析（记忆口诀）
        • 4、开始做题
        • 5、注意事项：
          • 1、单行的数据运算是平行宇宙，i++这样的副作用无法马上显现！
          • 2、( , , )这样的格式，返回的是最后一个值，本质是优先级的使用
          • 逗号运算符得到的结果都是括号最后一个表达式的值，C选项最后x+y的值应该是16，注意每个选项中的X+1其实是没有意义的，它没有重新赋值给X，所以经过这一步X的值是没有变化的。
          • 3、逻辑运算的结果，
          • 4、没有逗号表达式的“行”是平行宇宙，副作用不显现，但是“,”分隔的不是平行宇宙，副作用现象会发生

        # 八、sizeof真的是运算符

        下面程序的输出是:
#include <stdio.h>
int main()
{
    int i = 1;
    sizeof(i++);
    printf("%d\n", i);
    return 0;
}

B 4
C 2
D 8

        解析（复制别人的解释）：

        根据C99规范， sizeof是一个编译时刻就起效果的运算符，在其内的任何运算都没有意义， sizeof(i++); 在编译的时候被翻译成 sizeof((i++的数据类型)) 也就是 sizeof(int); 不会执行i++了。

        sizeof是运算符,它在编译时起作用，而不是运行时。 也就是说，在编译时，就得到了sizeof(int)，运行时，i++根本没有执行过。
