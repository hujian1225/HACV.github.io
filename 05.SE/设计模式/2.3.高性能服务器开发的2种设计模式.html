        # 01.维基百科

        • 反应器模式（Reactor_pattern）
        • Proactor模式

        来源：

        • 当下开源软件能做到网络高性能的原因就是 I/O 多路复用吗？
          • 是的，基本是基于 I/O 多路复用，用过 I/O 多路复用接口写网络程序的同学，肯定知道是面向过程的方式写代码的，这样的开发的效率不高。
          • 于是，大佬们基于『面向对象』的思想，对 I/O 多路复用作了一层封装，让使用者不用考虑底层网络 API 的细节，只需要关注应用代码的编写。
          • 大佬们还为这种模式取了个让人第一时间难以理解的名字：Reactor 模式。
          • 这里的反应指的是「对事件反应」，也就是来了一个事件，Reactor 就有相对应的反应/响应。事实上，Reactor 模式也叫 Dispatcher 模式，我觉得这个名字更贴合该模式的含义，即 I/O 多路复用监听事件，收到事件后，根据事件类型分配（Dispatch）给某个进程 / 线程。

        • Reactor模式分类

          • 单Reactor+多(进程/线程)

          • 多Reactor+多(进程/线程)
